Receiving Blank Pages

A given action results in a completely blank, white page

Troubleshooting Blank Pages

Blank pages in ExpressionEngine, where also the source is empty, are usually the result of a suppressed PHP error.

The first step is to ensure that error reporting is enabled:

  • Check PHP Info for display_errors and ensure it is set to "yes". If it is not then the host can change this setting
  • Check Admin -> System Preferences -> Output and Debugging Preferences to ensure that Debug Preferences are set to 1: PHP/SQL error messages shown only to Super Admins
  • In index.php change error_reporting(0) to error_reporting(E_ALL);
  • Check the server error logs for the cases where errors are output to logs and not to the screen

Following is a list of the more common circumstances that can result in a blank page:

  • Running out of PHP memory: This can occur especially when working with large images
  • Errors with Conditionals in the Templates.
  • Plugin Syntax errors when visiting Admin -> Utilities -> Plugin Manager
  • RSS templates not set to Template Type: RSS Template
  • Comment forms may output a blank page when using CAPTCHA if TrueType is enabled but not supported on the server. Go to Admin > Captcha Preferences and turn off the TrueType Font option.
